权威链 (Authority Chain)
L5 Gateway State
运行时状态权威。run metadata、current stage、artifact refs、pending decisions、resume checkpoints、events.jsonl 物理存储。可变，覆盖旧值。
L6 Hermes Kanban
任务生命周期权威。task existence、native status、parent/child dependencies、blocking、completion 的唯一直接来源。Gateway 通过 CLI 驱动但不替代。
L7 Audit
不可变证据权威。append-only 记录所有 decisions、retries、fallbacks、failures、downgrades、closeout。不可从 Event 重建。
L8 Artifacts
结构化产物权威。schema-valid、URI-addressed、revision 写新版本不覆盖。被 State 和 Audit 引用。
Recovery 原则
Recovery authority comes from Hermes Kanban lifecycle + Gateway State + State Artifacts + Audit Artifacts. Cache objects, worker summaries, and model self-report must not be used as resume authority.
非权威层 (Projection & Cache)
L9 Event Projection
观察投影，非 workflow truth。从 State/Audit/Kanban/Artifacts 重建。per-run monotonic seq。post-commit emit。inconsistent 时 Kimi resync。
L10 Cache
仅重建优化数据。debate_result、knowledge_summary、capabilities、gateway_projection。backend=local_filesystem。丢失 → degraded，不阻塞。
L3 Gateway Advancement Gate
Gate 不是权威存储，但是权威的校验门。Worker 输出必须通过 schema/write_scope/risk/evidence 校验才能转化为 State/Kanban/Audit 更新。
L1 Kimi
外部监督者。低/中风险下可 approve；高风险 Human Approval 不可覆盖。不是系统的一部分，但 decision authority 可覆盖 Gateway 默认流转。
层间依赖与数据流向
核心数据流（以 Create Run 为例）
L1 Kimi → L2 API → L3 Gateway → 先写 L4 Journal → 并行 L5 State L6 Kanban L7 Audit L8 Artifacts → 后写 L9 Event
L3 Gateway ↔ L10 Cache ↓ 调度 L11 Workers → 生成产物 → L8 Artifacts → 过 Gate → 更新 L5/L6/L7
