    - `TEST_RUNNER := docs/orchestra/scripts/tests/run-all.sh`
    - `RISK_TESTS := docs/orchestra/scripts/tests/test-risk-check.sh docs/orchestra/scripts/tests/test-risk-decisions.sh docs/orchestra/scripts/tests/test-decision-cli.sh`

    - `test`: depend on `test-unit test-risk lint-json lint-shell upstream-status`.

    - `test-risk`: loop over `$(RISK_TESTS)` and run `bash "$$script"` for each script with fail-fast behavior.
    - `lint-json`: run `find . -path './.git' -prune -o -name '*.json' -type f -print0 | xargs -0 -r -n1 python3 -m json.tool >/dev/null`.
    - `lint-shell`: if `shellcheck` is available, run shellcheck over these real paths only: `docs/orchestra/scripts/setup.sh`, `docs/orchestra/scripts/lib/*.sh`, `docs/orchestra/scripts/bin/orch-*`, `docs/orchestra/scripts/tests/*.sh`, and `docs/orchestra/scripts/tests/lib/*.sh`; otherwise print `shellcheck not found; skipping shell lint` and exit 0.
    - `upstream-status`: use Python stdlib to read `.planning/upstream/hermes-agent-pin.json` field `pin.commit`, print `repo pin: <commit>`, print `runtime path: <path>`, and then:
      - if `$(HERMES_AGENT_DIR)/.git` exists, run `git -C "$(HERMES_AGENT_DIR)" rev-parse HEAD`, print `runtime pin: <commit>`, print `status: match` and exit 0 when equal, or print `status: mismatch` and exit 1 when different;
      - if `$(HERMES_AGENT_DIR)/.git` does not exist, print `runtime pin: missing` and `status: runtime checkout not found; repo pin only`, then exit 0.

    Keep the Makefile ASCII-only, use tab-indented recipes, and avoid modifying existing shell test scripts.

| Boundary | Description |

| Makefile -> existing test scripts | Developer workflow delegates execution to existing repo scripts. |

| Makefile -> runtime Hermes checkout | `upstream-status` reads host-local runtime Git checkout state outside the repository. |
