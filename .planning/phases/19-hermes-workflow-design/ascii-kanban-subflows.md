### 任务状态机流转

```
                            ┌─────────────┐
                            │   用户提交   │
                            │   原始需求   │
                            └──────┬──────┘
                                   │
                                   ▼
   ┌─────────────────────────────────────────────────────────────────┐
   │  triage ──→ todo ──→ ready ──→ running ──→ blocked ──→ done   │
   │   (粗想法)    (具体)   (可执行)   (执行中)     (等输入)    (完成)  │
   └─────────────────────────────────────────────────────────────────┘
                                   │
        ┌──────────────────────────┼──────────────────────────┐
        │                          │                          │
        ▼                          ▼                          ▼
   ┌─────────┐              ┌─────────┐                 ┌─────────┐
   │   PM    │            │Dispatcher│                │  用户    │
   │ 拆解任务  │              │ 派发worker│               │ unblock │
   │ 创建子任务│              │ 监控执行  │               │ 做决策  │
   └─────────┘              └─────────┘                 └─────────┘
        │                          │                          │
        │                          │                          │
        ▼                          ▼                          ▼
   ┌─────────┐              ┌─────────┐                 ┌─────────┐
   │ todo    │─────────────→│ ready   │                 │ blocked │
   │ (排队等 │   parents    │ (可以干 │                 │ (暂停等 │
   │  依赖)  │   全done后   │  了)    │                 │  回复)  │
   └─────────┘              └─────────┘                 └─────────┘
        ▲                                               │
        │                                               │
        │         ┌─────────────┐                       │
        │         │   archived  │◄──────────────────────┘
        │         │   (归档)    │    done 后手动/GC
        │         └─────────────┘
        │               ▲
        └───────────────┘
```

**大白话：** 一张工厂看板，工单从左到右流动：粗想法→PM 拆解为具体任务→可以开工→正在干→卡住等回复→干完了→归档收起来。

### 任务依赖链（parents）

```
                         ┌──────────────────┐
                         │       PM         │
                         │  创建任务图      │
                         └────────┬─────────┘
                                  │
                    ┌─────────────┼─────────────┐
                    │             │             │
                    ▼             ▼             ▼
              ┌─────────┐  ┌─────────┐  ┌─────────┐
              │   T1    │  │   T2    │  │   T3    │
              │ 实现JWT │  │ 写测试  │  │ 审查代码│
              │ 模块    │  │         │  │         │
              │ assignee│  │ assignee│  │ assignee│
              │implementer│ │implementer│ │reviewer│
              │ parents=[]│ │parents=[T1]│ │parents=[T1]│
              └────┬────┘  └────┬────┘  └────┬────┘
                   │            │            │
                   │   T1 done  │            │
                   └─────┬──────┘            │
                         │                   │
                         ▼                   ▼
                    ┌─────────┐        ┌─────────┐
                    │  T2, T3 │        │  T2, T3 │
                    │ 同时变为 │        │ 同时变为 │
                    │  ready   │        │  ready   │
                    └────┬────┘        └────┬────┘
                         │                  │
                         ▼                  ▼
                    ┌─────────┐        ┌─────────┐
                    │Implementer│      │ Reviewer │
                    │ 跑 T2    │        │ 跑 T3    │
                    │ (写测试) │        │ (审代码) │
                    └────┬────┘        └────┬────┘
                         │                  │
                         │  T3 done         │
                         └─────┬────────────┘
                               │
                               ▼
                         ┌─────────┐
                         │   T4    │
                         │ 修复问题 │
                         │ assignee│
                         │implementer│
                         │parents=[T3]│
                         └────┬────┘
                              │
                              ▼
                         ┌─────────┐
                         │  T4 done │
                         │ 全部完成 │
                         └─────────┘


        T1 (实现JWT)
       /    \
      /      \
   T2 (测试)  T3 (审查)     ← T2, T3 并行，都等 T1
                \
                 \
                  T4 (修复)  ← T4 等 T3
```

**大白话：** 装修房子——水电做完才能刷墙和铺地板（并行），刷墙和铺地板都做完了才能装灯（串行）。系统自动管理这些先后关系。

### Dispatcher 工作循环

```
                        ┌─────────────┐
                        │  每 60 秒   │
                        │  循环开始   │◄────────────────────┐
                        └──────┬──────┘                     │
                               │                            │
                               ▼                            │
                    ┌─────────────────────┐                 │
                    │  步骤 1: 回收超时    │                 │
                    │  claim               │                 │
                    │  (心跳超时>2轮)      │                 │
                    └──────────┬──────────┘                 │
                               │                            │
                               ▼                            │
                    ┌─────────────────────┐                 │
                    │  步骤 2: 回收崩溃    │                 │
                    │  worker              │                 │
                    │  (PID 不存在)        │                 │
                    └──────────┬──────────┘                 │
                               │                            │
                               ▼                            │
                    ┌─────────────────────┐                 │
                    │  步骤 3: 提升任务    │                 │
                    │  todo → ready        │                 │
                    │  (parents 全 done)   │                 │
                    └──────────┬──────────┘                 │
                               │                            │
                               ▼                            │
                    ┌─────────────────────┐                 │
                    │  步骤 4: 原子 claim  │                 │
                    │  ready 任务          │                 │
                    │  (防止并发竞争)      │                 │
                    └──────────┬──────────┘                 │
                               │                            │
                               ▼                            │
                    ┌─────────────────────┐                 │
                    │  步骤 5: spawn worker│                 │
                    │  按 assignee 分配    │                 │
                    │  注入环境变量:       │                 │
                    │  HERMES_KANBAN_TASK  │                 │
                    │  HERMES_KANBAN_BOARD │                 │
                    └──────────┬──────────┘                 │
                               │                            │
                               ▼                            │
                    ┌─────────────────────┐                 │
                    │  Worker 进程启动     │                 │
                    │  只能看到:           │                 │
                    │  - 自己的任务        │                 │
                    │  - 自己的工具集      │                 │
                    └──────────┬──────────┘                 │
                               │                            │
                               │  执行任务...                │
                               │                            │
                               ▼                            │
                    ┌─────────────────────┐                 │
                    │  等待 60 秒...       │─────────────────┘
                    │  下一轮循环          │
                    └─────────────────────┘
```

**大白话：** PM 负责拆解需求、创建任务图和分配责任人。调度员（Orchestrator）是个"每分钟看一次全局"的路由器：有没有假死的工人？有没有真死的？前面的活干完了后面能开始了吗？能开始的任务派给谁？派的时候给工人发工牌（环境变量），告诉他"你只许看这个任务、用这些工具"。

### Handoff 机制（任务交接）

```
   ┌─────────────────────┐           ┌─────────────────────┐
   │   Worker A          │           │   Worker B          │
   │  (Implementer)      │           │  (Implementer)      │
   │  完成任务 T1        │           │  接任务 T4          │
   └──────────┬──────────┘           └──────────┬──────────┘
              │                                  │
              │  kanban_complete()               │  kanban_show()
              │                                  │
              ▼                                  ▼
   ┌─────────────────────────────────────────────────────────────┐
   │                    Kanban Board (SQLite)                    │
   │  ┌───────────────────────────────────────────────────────┐  │
   │  │  Task T1 (done)                                       │  │
   │  │  ───────────────────────────────────────────────────  │  │
   │  │  summary: "JWT 认证已实现 (TDD)"                       │  │
   │  │  metadata:                                            │  │
   │  │  {                                                    │  │
   │  │    "behaviors": [...],                                │  │
   │  │    "regression": {"run": 14, "passed": 14},          │  │
   │  │    "changed_files": ["auth/jwt.py"],                  │  │
   │  │    "decisions": ["用 RS256"],                         │  │
   │  │    "pitfalls": ["token刷新用滑动窗口"]                 │  │
   │  │  }                                                    │  │
   │  └───────────────────────────────────────────────────────┘  │
   │                                                             │
   │  ┌───────────────────────────────────────────────────────┐  │
   │  │  Task T4 (running)                                    │  │
   │  │  ───────────────────────────────────────────────────  │  │
   │  │  parents: [T1, T3]                                    │  │
   │  │  读取 parent handoff → 获取 T1/T3 的 metadata         │  │
   │  └───────────────────────────────────────────────────────┘  │
   └─────────────────────────────────────────────────────────────┘
```

**大白话：** 晚班工人下班时写一张交接单（改了什么文件、测试过没过、做了什么决策、踩了什么坑），贴在任务板上。早班工人接班时读这张单子，不需要从头翻代码就知道前面发生了什么。

### Worker 崩溃状态回滚

```
                         ┌─────────────┐
                         │  任务创建   │
                         │  状态: todo │
                         └──────┬──────┘
                                │
                                ▼
                    ┌─────────────────────────┐
                    │  parents 全 done        │
                    │  状态: → ready           │
                    └───────────┬─────────────┘
                                │
                                ▼
                    ┌─────────────────────────┐
                    │  Dispatcher spawn       │
                    │  Worker 前：            │
                    │  拍快照!                │
                    │  git stash push         │
                    │  -m "pre-task:T1"       │
                    └───────────┬─────────────┘
                                │
                                ▼
                    ┌─────────────────────────┐
                    │  Worker 开始执行任务     │
                    │  状态: → running         │
                    │                         │
                    │  ┌─────────────────────┐│
                    │  │ 修改文件...         ││
                    │  │ 写代码...           ││
                    │  │ 运行测试...         ││
                    │  └─────────────────────┘│
                    └───────────┬─────────────┘
                                │
                    ┌───────────┴───────────┐
                    ▼                       ▼
            ┌─────────────┐         ┌─────────────┐
            │   正常完成   │         │   崩溃检测   │
            │ complete()  │         │             │
            └──────┬──────┘         └──────┬──────┘
                   │                       │
                   ▼                       │
            ┌─────────────┐               │
            │  状态: done  │               │  PID 不存在?
            │  Handoff     │               │  心跳超时?
            └─────────────┘               │
                                          ▼
                              ┌─────────────────────────┐
                              │  崩溃! 触发回滚         │
                              │                         │
                              │  1. 状态 → ready        │
                              │  2. git stash pop       │
                              │     恢复干净 workspace   │
                              │  3. rollback_count++    │
                              │  4. 记录审计日志         │
                              └───────────┬─────────────┘
                                          │
                                          ▼
                              ┌─────────────────────────┐
                              │  重新派发新 Worker       │
                              │  从干净状态重新开始      │
                              └─────────────────────────┘
```

**大白话：** 工人开始干活前系统自动"存档"。如果工人干到一半电脑蓝屏了，系统读档重来——下一个工人看到的是和上一个工人开始时一模一样的干净环境，不会被烂摊子困扰。

### 背压感知任务准入

```
                    ┌─────────────────────────┐
                    │  Dispatcher 轮询开始     │
                    └───────────┬─────────────┘
                                │
                                ▼
                    ┌─────────────────────────┐
                    │  计算背压比率            │
                    │                         │
                    │  ratio = impl_ready     │
                    │         ─────────────   │
                    │         max(rev_ready,1)│
                    └───────────┬─────────────┘
                                │
                    ┌───────────┴───────────┐
                    ▼                       ▼
            ┌─────────────┐         ┌─────────────┐
            │  rev_ready=0 │         │  rev_ready>0 │
            │  (审查没人)  │         │  (审查有积压)│
            └──────┬──────┘         └──────┬──────┘
                   │                       │
                   ▼                       ▼
            ┌─────────────┐         ┌─────────────┐
            │ ratio = ∞   │         │ ratio = N   │
            │ → 暂停派发  │         │ → 判断区间  │
            └──────┬──────┘         └──────┬──────┘
                   │                       │
                   │         ┌─────────────┼─────────────┐
                   │         ▼             ▼             ▼
                   │   ┌─────────┐  ┌─────────┐  ┌─────────┐
                   │   │ratio≤2  │  │2<ratio≤4│  │ ratio>4 │
                   │   │(正常)   │  │(降速)   │  │(暂停)   │
                   │   └────┬────┘  └────┬────┘  └────┬────┘
                   │        │            │            │
                   │        ▼            ▼            ▼
                   │   ┌─────────┐ ┌─────────┐ ┌─────────┐
                   │   │正常spawn│ │隔轮spawn│ │暂停spawn│
                   │   │全部ready│ │50% ready│ │0 ready  │
                   │   └────┬────┘ └────┬────┘ └────┬────┘
                   │        │           │           │
                   └────────┴───────────┴───────────┘
                                            │
                                            ▼
                              ┌─────────────────────────┐
                              │  等待 reviewer 消化积压  │
                              │  ratio 降下来后恢复      │
                              └─────────────────────────┘
```

**大白话：** 流水线的自动调速阀。审查工位堆了 20 个任务、写代码的只有 2 个——系统踩刹车：暂停招新的写代码工人。等审查那边把积压清到合理比例，再恢复招工。
