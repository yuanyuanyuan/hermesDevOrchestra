### 三层架构（知识资产管理）

```
        ┌─────────────────────────────────────────────┐
        │  ┌─────────────────────────────────────────┐│
        │  │           锁定层（不可自动修改）          ││
        │  │                                         ││
        │  │  ● Bundled skills (Hermes 内置)         ││
        │  │  ● Hub-installed skills (skills.sh)     ││
        │  │  ● Pinned skills (用户手动固定)          ││
        │  │                                         ││
        │  │  🔒 Curator 永远不碰                    ││
        │  │     只能用户手动修改                     ││
        │  └─────────────────────────────────────────┘│
        │                    ▲                        │
        │                    │ 审核后晋升              │
        │  ┌─────────────────────────────────────────┐│
        │  │           可变层（Agent 可修改）          ││
        │  │                                         ││
        │  │  ● Agent-created skills                 ││
        │  │  ● MEMORY.md (2200字符上限)             ││
        │  │  ● USER.md (1375字符上限)               ││
        │  │  ● Profile SOUL.md (用户审核后)         ││
        │  │                                         ││
        │  │  ✏️ Curator 定期审查清理                 ││
        │  │     合并、精简、归档过时内容             ││
        │  └─────────────────────────────────────────┘│
        │                    ▲                        │
        │                    │ 只增不改               │
        │  ┌─────────────────────────────────────────┐│
        │  │           事实表（只增不改）              ││
        │  │                                         ││
        │  │  ● Kanban task_runs + metadata          ││
        │  │  ● Session SQLite (FTS5 可搜索)         ││
        │  │  ● .learnings/ 目录 (经验分类归档)      ││
        │  │                                         ││
        │  │  📋 历史记录，任何人不能修改             ││
        │  │     只读、可搜索、可审计                 ││
        │  └─────────────────────────────────────────┘│
        └─────────────────────────────────────────────┘

        类比:
        ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
        │   公司规章   │    │   部门SOP   │    │   会议记录  │
        │  (只能老板改)│    │ (可优化更新)│    │  (永久存档) │
        │             │    │             │    │             │
        │ 锁定层      │───→│  可变层     │───→│  事实表     │
        │             │    │             │    │             │
        │ 不可自动修改│    │ Curator打理 │    │ 只增不改   │
        └─────────────┘    └─────────────┘    └─────────────┘
```

**大白话：** 公司有三类知识资产：规章制度（只能老板改）、部门经验笔记（AI 可以补充优化）、历史记录（永久存档不能动）。

### 实时层进化

```
   ┌─────────────────────────────────────────────────────────────────┐
   │                      Worker 执行任务中                           │
   └─────────────────────────────┬───────────────────────────────────┘
                                 │
              ┌──────────────────┼──────────────────┐
              │                  │                  │
              ▼                  ▼                  ▼
   ┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
   │  命令失败       │ │  用户纠正       │ │  发现新工作流   │
   │  npm install    │ │  "别这么写，    │ │  5+ tool calls  │
   │  报错 exit 1    │ │  要那样写"      │ │  形成固定模式   │
   └───────┬─────────┘ └───────┬─────────┘ └───────┬─────────┘
           │                   │                   │
           ▼                   ▼                   ▼
   ┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
   │ memory add      │ │ memory add      │ │ skill_manage    │
   │ "缺xxx包，      │ │ "用户偏好:      │ │ create          │
   │  下次先安装"    │ │  用方案B"       │ │ "xxx-checklist" │
   └───────┬─────────┘ └───────┬─────────┘ └───────┬─────────┘
           │                   │                   │
           └───────────────────┼───────────────────┘
                               │
                               ▼
                    ┌─────────────────────────┐
                    │   写入可变层             │
                    │   MEMORY.md / Skills     │
                    │   (实时学习，边干边记)    │
                    └─────────────────────────┘
```

**大白话：** 工人一边干活一边长记性——出错了记一下怎么修、用户纠正了记一下偏好、发现每次都要走5步就写成 checklist 以后直接用。

```
                    ┌─────────────┐
                    │   每 7 天   │◄────────────────────┐
                    │   自动触发  │                     │
                    └──────┬──────┘                     │
                           │                            │
                           ▼                            │
            ┌──────────────────────────────┐            │
            │      Curator 审查周期         │            │
            └──────────────┬───────────────┘            │
                           │                            │
           ┌───────────────┼───────────────┐            │
           ▼               ▼               ▼            │
   ┌─────────────┐ ┌─────────────┐ ┌─────────────┐     │

   │   skills    │ │   skills    │ │   改进      │     │
   │             │ │             │ │             │     │

   └──────┬──────┘ └──────┬──────┘ └──────┬──────┘     │
          │               │               │            │

        ┌─────────────────┐
```

**大白话：** 每季度自动做"文档大扫除"——半年没用的归档、两份讲同一件事的合并、写满了就精简一下。还会每周复盘"最近有什么模式可以提炼成经验"。

### 分层经验归档（Hierarchical Learnings）

```
                    .learnings/
                    │
        ┌───────────┼───────────┐
        │           │           │
        ▼           ▼           ▼
   ┌─────────┐ ┌─────────┐ ┌─────────┐
   │project-a│ │_global/ │ │_pending/│
   │/        │ │         │ │         │
   │         │ │ 经审核  │ │ 待审核  │
   │ 默认写入 │ │ 跨项目  │ │ 晋升请求│
   │ 私有经验 │ │ 共享    │ │         │
   └────┬────┘ └────┬────┘ └────┬────┘
        │           │           │
        │           │           │
        │    ┌──────┴──────┐    │
        │    │             │    │
        │    ▼             ▼    │
        │ ┌─────────┐  ┌─────────┐
        │ │ 显式标记 │  │ curator │
        │ │cross-project│ │ 审核    │
        │ │ + quality  │  │ score≥60│
        │ │  score≥60  │  │         │
        │ └────┬────┘  └────┬────┘
        │      │            │
        └──────┼────────────┘
               │
               ▼
        ┌─────────────┐
        │  进入 _global/ │
        │  所有项目可用   │
        └─────────────┘

        查询优先级:
        ┌─────────────────────────────────────────────┐
        │                                             │
        │   Agent 查询 "Webpack 配置"                  │
        │        │                                    │
        │        ▼                                    │
        │   ┌─────────────────┐                       │
        │   │ 1. 查项目命名空间 │ ← 优先使用           │
        │   │    project-a/    │   本项目的经验        │
        │   └────────┬────────┘                       │
        │            │                                │
        │            │ 未找到?                        │
        │            ▼                                │
        │   ┌─────────────────┐                       │
        │   │ 2. 查全局命名空间 │ ←  fallback         │
        │   │    _global/      │                       │
        │   └────────┬────────┘                       │
        │            │                                │
        │            │ 矛盾?                          │
        │            ▼                                │
        │   ┌─────────────────┐                       │
        │   │ 3. 返回冲突警告  │ ← <conflict_warning>  │
        │   │  项目 vs 全局   │   两边都显示           │
        │   └─────────────────┘                       │
        │                                             │
        └─────────────────────────────────────────────┘
```

**大白话：** 经验默认是"部门内部资料"。特别好的可以申请"升级为公司级资料"，经知识管家审核后所有项目都能参考。但查的时候优先查本项目的——你自己的踩坑记录比公司通用文档更相关。
