### 实时问答流程（tmux 方式）

```
   ┌─────────────────────────────────────────────────────────────────┐
   │                       tmux 会话网格                              │
   │                                                                 │
   │  ┌───────────────────────┐    ┌───────────────────────┐        │
   │  │  Session:             │    │  Session:             │        │
   │  │  hermes-alpha-impl-T1 │    │  hermes-alpha-rev-T3  │        │
   │  │                       │    │  (按需创建，用完即毁)  │        │
   │  │  Implementer Worker   │    │                       │        │
   │  │  正在写代码...         │    │  Reviewer (claude -p) │        │
   │  │                       │    │  一次性调用           │        │
   │  │  [屏幕输出]           │    │  [给出决策]           │        │
   │  │  "该用 RS256 还是     │    │  "用 RS256"          │        │
   │  │   HS256?"             │    │                       │        │
   │  └───────────┬───────────┘    └───────────┬───────────┘        │
   │              │                              │                   │
   │              │  ① capture-pane 读取疑问    │                   │
   │              │◄─────────────────────────────┘                   │
   │              │                              (Hermes 做路由器)   │
   │              │  ② send-keys 传递决策        │                   │
   │              └────────────────────────────►│                   │
   │                                                                 │
   └─────────────────────────────────────────────────────────────────┘

        步骤详解:
        ┌─────────┐     ┌─────────┐     ┌─────────┐
        │Worker   │     │ Hermes  │     │Reviewer │
        │输出疑问 │────→│capture  │────→│-p 调用  │
        │到 tmux  │     │读取    │     │给出决策 │
        └─────────┘     └─────────┘     └─────────┘
                                              │
                                              ▼
                                        ┌─────────┐
                                        │ 决策结果│
                                        │ 回传    │
                                        └────┬────┘
                                             │
                                             ▼
                                        ┌─────────┐
                                        │ Hermes  │
                                        │send-keys│
                                        │回 Worker│
                                        └────┬────┘
                                             │
                                             ▼
                                        ┌─────────┐
                                        │ Worker  │
                                        │ 继续执行│
                                        └─────────┘
```

**大白话：** 两个 AI 工人在同一个 tmux 会议室工作。A 写到一半问"用方案 A 还是 B？"，Hermes 听到了转头问专家 B，得到答案后传回给 A，A 继续干活。

### Agent 间通信可靠性方案（推荐：方案 A）

```
   ┌─────────────────────────────────────────────────────────────────┐
   │                     Kanban Board (SQLite)                        │
   │  ┌─────────────────────────────────────────────────────────┐    │
   │  │  Task T1 (blocked)                                      │    │
   │  │  ───────────────────────────────────────────────────    │    │
   │  │  status: blocked                                        │    │
   │  │  reason: "reviewer-needed: 用 RS256 还是 HS256?"        │    │
   │  │  block_time: 2026-05-10T10:00:00Z                       │    │
   │  └─────────────────────────────────────────────────────────┘    │
   │                              │                                  │
   │  ┌───────────────────────────┼─────────────────────────────┐    │
   │  │                           ▼                             │    │
   │  │  Task T1-rev (ready→running)                            │    │
   │  │  ───────────────────────────────────────────────────    │    │
   │  │  title: "审查: T1 技术决策"                              │    │
   │  │  assignee: reviewer                                     │    │
   │  │  body: "T1 需要决策: 用 RS256 还是 HS256?"               │    │
   │  │  priority: high  ← 自动高优先级                         │    │
   │  └─────────────────────────────────────────────────────────┘    │
   │                              │                                  │
   │                              ▼                                  │
   │  ┌─────────────────────────────────────────────────────────┐    │
   │  │  Task T1-rev (done)                                     │    │
   │  │  ───────────────────────────────────────────────────    │    │
   │  │  summary: "建议用 RS256"                                 │    │
   │  │  metadata:                                              │    │
   │  │  { "decision": "RS256", "reason": "支持 key rotation" } │    │
   │  └─────────────────────────────────────────────────────────┘    │
   │                              │                                  │
   │                              ▼                                  │
   │  ┌─────────────────────────────────────────────────────────┐    │
   │  │  Task T1 (unblocked)                                    │    │
   │  │  ───────────────────────────────────────────────────    │    │
   │  │  status: ready → running                                │    │
   │  │  metadata.decision: "RS256"  ← 决策写入                 │    │
   │  └─────────────────────────────────────────────────────────┘    │
   └─────────────────────────────────────────────────────────────────┘

        时序图:
        Implementer          Dispatcher          Reviewer
             │                   │                  │
             │ ① block()         │                  │
             │ "reviewer-needed" │                  │
             │──────────────────►│                  │
             │                   │ ② 检测前缀        │
             │                   │ 创建 reviewer task│
             │                   │─────────────────►│
             │                   │                  │ ③ 分析并
             │                   │                  │    complete()
             │                   │◄─────────────────│
             │                   │ ④ unblock +     │
             │                   │    写入决策       │
             │◄──────────────────│                  │
             │ ⑤ 读取 handoff    │                  │
             │    继续执行        │                  │
             │                   │                  │
```

**大白话：** 不用喊话（tmux），改用工单系统（Kanban）。A 遇到问题了填一张"求助工单"，系统自动派 B 来处理，处理完把结论写回工单，A 看到后继续干活。消息不会丢，有记录可查。

### Tmux Session 预热池

```
                    ┌─────────────────────────┐
                    │     预热池 (Warm Pool)   │
                    │    默认 2-3 个 session   │
                    │                         │
                    │  ┌─────────────────┐    │
                    │  │ hermes-warm-1   │    │
                    │  │ (预加载环境)     │    │
                    │  │ - HERMES_KANBAN │    │
                    │  │   _WORKSPACE    │    │
                    │  │ - git config    │    │
                    │  │ - common env    │    │
                    │  └─────────────────┘    │
                    │  ┌─────────────────┐    │
                    │  │ hermes-warm-2   │    │
                    │  │ (预加载环境)     │    │
                    │  └─────────────────┘    │
                    └───────────┬─────────────┘
                                │
              ┌─────────────────┴─────────────────┐
              │                                   │
              ▼                                   ▼
   ┌─────────────────────┐           ┌─────────────────────┐
   │  Dispatcher 需要     │           │  Dispatcher 需要     │
   │  spawn worker        │           │  spawn worker        │
   └──────────┬──────────┘           └──────────┬──────────┘
              │                                   │
              ▼                                   ▼
   ┌─────────────────────┐           ┌─────────────────────┐
   │  ① 从池中 claim     │           │  ① 从池中 claim     │
   │     warm session     │           │     warm session     │
   └──────────┬──────────┘           └──────────┬──────────┘
              │                                   │
              ▼                                   ▼
   ┌─────────────────────┐           ┌─────────────────────┐
   │  ② 注入环境变量      │           │  ② 注入环境变量      │
   │     HERMES_KANBAN_  │           │     HERMES_KANBAN_  │
   │     TASK/BOARD      │           │     TASK/BOARD      │
   └──────────┬──────────┘           └──────────┬──────────┘
              │                                   │
              ▼                                   ▼
   ┌─────────────────────┐           ┌─────────────────────┐
   │  ③ 重命名 session   │           │  ③ 重命名 session   │
   │  hermes-alpha-      │           │  hermes-beta-       │
   │  impl-t_abc123      │           │  rev-t_def456       │
   └──────────┬──────────┘           └──────────┬──────────┘
              │                                   │
              ▼                                   ▼
   ┌─────────────────────┐           ┌─────────────────────┐
   │  Worker 开始干活     │           │  Worker 开始干活     │
   │  (毫秒级启动)        │           │  (毫秒级启动)        │
   └──────────┬──────────┘           └──────────┬──────────┘
              │                                   │
              └───────────┬───────────────────────┘
                          │
                          ▼
               ┌─────────────────────┐
               │  ④ 异步补充新 warm   │
               │     session 到池中   │
               │     (保持池大小)     │
               └─────────────────────┘
```

**大白话：** 餐厅提前备好干净的碗，客人来了直接盛菜。工人来干活时不用现配环境，直接进一个"热好的"session，换上工牌就开始干。
