> **架构说明（2026-05-11 更新）：** 本文档中的 "worker 进程" 是 Hermes Profile 进程（轻量编排层），通过 JSON 协议委托给外部 CLI 引擎执行实际工作。详见 [EXTERNAL-CLI-ENGINE.md](./EXTERNAL-CLI-ENGINE.md)。

## 一、核心业务流程（Key Flows F0–F7）

> 📎 **相关文档**：
> - F0 → Phase 0 平台能力确认
> - F1 → 需求提交与澄清（PM）
> - F2 → 技术研判：Research + POC（Researcher / Implementer）
> - F3 → 任务拆解与派发（PM / Orchestrator）
> - F4 → 多项目并行任务生命周期（含背压）
> - F5 → L3 风险升级与用户决策
> - F6 → 故障处理与 SRE 分析（人工升级触发）
> - F7 → 完成通知与用户审核
>
> **能力来源说明：** F0（Phase 0 验证）、F2 中的 Dispatcher 循环/Board 隔离属于 `[Hermes 官方]`。F2 中的背压感知（R5）、F5 中的 L1/L2/L3 风险分级和 Risk Policy YAML、F4 中的 SRE-Observer 人工升级触发属于 `[Phase 19 增量]`。L1/L2/L3 为项目设计概念，非 Hermes 官方机制（RFC #16102 支持 user-space 构建）。
