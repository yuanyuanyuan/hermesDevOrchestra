### L3 升级流程（完整路径）

```
                         ┌─────────────┐
                         │  Worker     │
                         │  执行任务   │
                         └──────┬──────┘
                                │
                                ▼
                    ┌─────────────────────────┐
                    │  遇到需要做决策的场景    │
                    │                         │
                    │  ① 代码实现细节?        │
                    │  ② 技术方案/API设计?    │
                    │  ③ 安全凭证/破坏性重构? │
                    └───────────┬─────────────┘
                                │
                    ┌───────────┼───────────┐
                    ▼           ▼           ▼
            ┌───────────┐ ┌───────────┐ ┌───────────┐
            │  场景 ①   │ │  场景 ②   │ │  场景 ③   │
            │ 变量命名  │ │ API设计   │ │ git push  │
            │ 代码风格  │ │ 技术选型  │ │ --force   │
            │ 错误处理  │ │ 架构调整  │ │ 删数据库  │
            └─────┬─────┘ └─────┬─────┘ └─────┬─────┘
                  │             │             │
                  ▼             ▼             ▼
            ┌───────────┐ ┌───────────┐ ┌───────────┐
            │   L1      │ │   L2      │ │   L3      │
            │ Worker    │ │ Reviewer  │ │   CEO     │
            │ 自行决定  │ │ Hard Gate │ │ (用户)    │
            │ (秒级)    │ │ 严格只读  │ │ 决策      │
            └─────┬─────┘ └─────┬─────┘ └─────┬─────┘
                  │             │             │
                  ▼             ▼             ▼
            ┌───────────┐ ┌───────────┐ ┌───────────┐
            │ 直接执行  │ │ claude -p │ │ kanban_   │
            │ 记录决策  │ │ 一次性调用│ │ block()   │
            │ 到metadata│ │ Reviewer  │ │           │
            │           │ │ 给出决策  │ │ 等待用户  │
            └───────────┘ └─────┬─────┘ │  unblock  │
                                │       └─────┬─────┘
                                │             │
                                ▼             ▼
                          ┌─────────────────────────┐
                          │  决策记录到任务 metadata  │
                          │  审计日志归档             │
                          └─────────────────────────┘

        L3 详细时序:
        ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
        │ Worker  │    │Dispatcher│   │ Risk    │    │  CEO    │
        │         │    │         │    │ Policy  │    │ (User)  │
        └────┬────┘    └────┬────┘    └────┬────┘    └────┬────┘
             │              │              │              │
             │ 执行危险命令  │              │              │
             │─────────────►│              │              │
             │              │ 匹配 policy  │              │
             │              │─────────────►│              │
             │              │              │ L3 匹配!     │
             │              │◄─────────────│              │
             │              │              │              │
             │  block()     │              │              │
             │◄─────────────│              │              │
             │  (任务暂停)  │              │              │
             │              │  Gateway推送  │              │
             │              │────────────────────────────►│
             │              │              │              │
             │              │              │         ┌────┴────┐
             │              │              │         │[同意]   │
             │              │              │         │[拒绝]   │
             │              │              │         │[需信息] │
             │              │              │         └────┬────┘
             │              │              │              │
             │              │◄─────────────────────────────│
             │              │  unblock + 决策              │
             │  resume      │                              │
             │◄─────────────│                              │
             │  按决策执行  │                              │
             │              │                              │
```

**大白话：** 小事工人自己定（改变量名），中等事问 Reviewer（严格只读的 Hard Gate，只审不改），大事必须老板签字（删数据库）。大事不会因为你没看手机就自动通过。

### 声明式风险策略引擎

```
                    ┌─────────────────────────┐
                    │  policies/risk.yaml     │
                    │  （集中配置文件）        │
                    └───────────┬─────────────┘
                                │
                                ▼
   ┌─────────────────────────────────────────────────────────────────┐
   │                                                                 │
   │  policies:                                                      │
   │  ─────────────────────────────────────────────────────────────  │
   │  - pattern: "rm -rf /*"                                         │
   │    level: L3                                                    │
   │    approver: user                                               │
   │    timeout: 0    ← 永不自动通过                                 │
   │                                                                 │
   │  - pattern: "git push --force"                                  │
   │    level: L3                                                    │
   │    approver: user                                               │
   │    timeout: 300  ← 5分钟无响应则escalate                        │
   │                                                                 │
   │  - pattern: "修改 CI/CD 配置"                                    │
   │    level: L2                                                    │
   │    approver: reviewer                                           │
   │    timeout: 600                                                 │
   │                                                                 │
   │  - pattern: "变量重命名"                                         │
   │    level: L1                                                    │
   │    approver: self                                               │
   │                                                                 │
   └─────────────────────────────────────────────────────────────────┘
                                │
              ┌─────────────────┼─────────────────┐
              ▼                 ▼                 ▼
   ┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
   │ ① Orchestrator  │ │ ② Worker SOUL   │ │ ③ Sentinel      │
   │   spawn 前注入   │ │   .md 引用       │ │   统一拦截       │
   │   HERMES_RISK_   │ │   做决策依据     │ │   消费           │
   │   POLICY 环境变量│ │                 │ │                 │
   └─────────────────┘ └─────────────────┘ └─────────────────┘
                                │
                                ▼
                    ┌─────────────────────────┐
                    │  所有 Worker 统一消费    │
                    │  避免规则散落在 prompt 中│
                    │  一处修改，全局生效      │
                    └─────────────────────────┘
```

**大白话：** 公司规章制度写成一张"自动检查清单"，所有 AI 工人干活前先过一遍：删根目录→必须老板签字，改 CI 配置→问技术负责人，变量重命名→自己定。一处修改，所有工人都遵守。
