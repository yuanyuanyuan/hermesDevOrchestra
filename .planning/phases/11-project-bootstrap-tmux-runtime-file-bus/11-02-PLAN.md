# Phase 11 Plan 02: tmux Runtime Lifecycle & Codex Task Dispatch

## Objective

- **What:** Implement reusable Claude/Codex tmux session startup, project watcher lifecycle, and `task.md` dispatch into Codex.
- **Why:** Phase 11 success requires user tasks to reach Codex through the per-project file bus while preserving tmux isolation and multi-project non-blocking behavior.
- **Output:** Runtime-aware `orch-start`, `orch-stop`, internal `orch-bus-loop`, and detailed status integration.

## Threat Model

<threat_model>

| ID | Threat | Severity | Mitigation |
|----|--------|----------|------------|
| T-11-04 | `orch-start` kills healthy sessions and loses in-progress work | high | Use `tmux has-session` plus pane health checks; only recreate absent/dead sessions. |
| T-11-05 | Codex runs with unrestricted dangerous bypass flags | high | Use `--full-auto`; explicitly grep-ban `--dangerously-bypass-approvals-and-sandbox`. |
| T-11-06 | Watcher dispatches duplicate tasks on every poll | medium | Store current `task_id`/content hash in State and skip already-dispatched task content. |
| T-11-07 | One blocked project monopolizes scheduler loop | medium | Per-project watcher records `waiting`/`blocked` and sleeps/yields instead of busy-looping. |

</threat_model>

    - Use `inotifywait -q -e create,modify,move "$RUNTIME_DIR"` when available; otherwise sleep 2 seconds between scans.
    - Implement `--once` mode for fixture tests: process current bus state one time and exit.

      - write current state `executing`
      - write the hash to `"$STATE_DIR/last-task.hash"`
      - send this command into the Codex tmux shell with `tmux send-keys -t "$CODEX_SESSION" ... Enter`:
        `cat "$RUNTIME_DIR/task.md" | codex exec --full-auto --json --output-last-message "$RUNTIME_DIR/codex-result.md" - > "$STATE_DIR/codex-events.jsonl" 2>> "$STATE_DIR/codex.err"`
    - The Codex prompt wrapper must instruct Codex to write `codex-question.md` and stop if it cannot proceed, and to produce a final JSON envelope in `codex-result.md` on completion.
    - Never use `--dangerously-bypass-approvals-and-sandbox`.
    - README documents `notification_hook` or Codex `notify` only as optional acceleration that may touch `.codex-done`; file polling and `codex-result.md` remain canonical.
