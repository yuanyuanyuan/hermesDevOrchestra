  <action>
    Extend `orch-bus-loop` with question and decision routing:
    - When `codex-question.md` exists and `claude-decision.md` does not exist, validate that the question has `project_id` matching `"$PROJECT_ID"` when the key is present.
    - If validation fails, write a warning to `"$STATE_DIR/orch-bus-loop.log"` and do not route the file.
    - Write state `waiting`.
    - Send this command into `"$CLAUDE_SESSION"` with `tmux send-keys`:
      `cat "$RUNTIME_DIR/codex-question.md" | claude -p --output-format json --permission-mode auto "You are Claude Supervisor for project [$PROJECT_ID]. Read the Codex question from stdin and write a JSON decision envelope for claude-decision.md. Do not approve L3/L4; write escalation guidance instead." > "$RUNTIME_DIR/claude-decision.md" 2>> "$STATE_DIR/claude.err"`
    - The decision envelope must include `schema_version`, `message_id`, `project_id`, `task_id`, `correlation_id`, `status`, `author`, `authority`, `timestamp`, `decision`, `rationale`, and `execution.authority_sufficient`.
    - If `escalation.md` appears at any point, write project state `blocked` and do not send more commands to Codex.
    - When `claude-decision.md` exists, `codex-question.md` exists, and no `codex-result.md` exists, validate matching `project_id` and `correlation_id` across `task.md`, `codex-question.md`, and `claude-decision.md` when those keys are present.
    - If the decision has `execution.authority_sufficient` set to `false`, write state `blocked` and do not continue Codex.
    - If sufficient, write state `executing`.
    - Continue Codex with a fresh one-shot invocation, not session continuation, to honor D-11-04:
      `cat "$RUNTIME_DIR/task.md" "$RUNTIME_DIR/claude-decision.md" | codex exec --full-auto --json --output-last-message "$RUNTIME_DIR/codex-result.md" - >> "$STATE_DIR/codex-events.jsonl" 2>> "$STATE_DIR/codex.err"`
    - After `codex-result.md` appears, move `codex-question.md` to `"$AUDIT_DIR/pending/codex-question.$(date +%s).md"` to prevent repeated decision routing.
  </action>
