## Threat Model

<threat_model>

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User/agent text -> `orch-risk-check` | Untrusted operation strings and JSON envelope text cross into rule evaluation. |
| Runtime/State -> Audit | Volatile or mutable event context becomes durable JSONL evidence. |
| Package defaults -> User config | Default `rules.json` is copied into user-owned config without overwriting local edits. |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Tampering | `rules.json` install path | mitigate | `setup.sh` copies defaults only when `~/.hermes-orchestra/rules.json` is absent and never silently overwrites user config. |
| T-12-02 | Repudiation | `audit.jsonl` | mitigate | `orch_append_audit` writes timestamped JSON records with project, type, decision, approval_id, task_id, escalation_id, agent_source, and session_id fields. |
| T-12-03 | Information Disclosure | `orch-audit` output | mitigate | `orch-audit` defaults to last 20 records and supports `--level` filtering; it does not print secret-specific environment variables. |
| T-12-04 | Denial of Service | Audit growth | mitigate | `orch_rotate_audit_if_needed` rotates `audit.jsonl` once it exceeds 10485760 bytes. |

</threat_model>
