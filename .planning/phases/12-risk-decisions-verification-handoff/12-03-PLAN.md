## Threat Model

<threat_model>

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Temporary HOME -> installer/helpers | Fixtures run against disposable roots and must not write persistent host state. |
| Fixture output -> reviewer trust | Smoke summaries become evidence for handoff and verification. |
| Documentation -> release confidence | Grep fixtures verify docs and coverage claims match implementation boundaries. |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-10 | Tampering | Host HOME or `/tmp` | mitigate | `assert.sh` helpers require tests to use `mktemp -d`, caller-owned cleanup, and explicit temp roots. |
| T-12-11 | Repudiation | Smoke results | mitigate | `run-all.sh` prints each test path, PASS/FAIL, and a final summary with non-zero failure status. |
| T-12-12 | Information Disclosure | Test logs | mitigate | Assertions print expected/actual values and log excerpts only from temp fixture directories. |
| T-12-13 | Spoofing | Coverage claims | mitigate | `test-docs.sh` checks exact helper names, deferred scope, and README linkage to `docs/COVERAGE-MATRIX.md`. |

</threat_model>
