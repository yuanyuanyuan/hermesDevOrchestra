## Threat Model

<threat_model>

### Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Fake PATH -> helper scripts | Tests inject fake `hermes`, `tmux`, `claude`, and `codex` commands to avoid live credentials. |
| Temporary HOME -> installer/helpers | Fixtures run against disposable user roots and must not write persistent host state. |
| Agent-authored decision text -> risk gate | `claude-decision.md` content can understate risk and must be rechecked by Hermes. |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-19 | Spoofing | Fake CLI fixtures | accept | Fixtures intentionally spoof external CLIs; fake commands stay under `mktemp -d` and PATH is scoped to each script. |
| T-12-20 | Tampering | Host HOME or `/tmp` | mitigate | Every fixture creates `TEST_HOME="$(mktemp -d)"`, sets `RUNTIME_ROOT="$TEST_HOME/tmp/hermes-orchestra"`, and cleans up with `trap`. |
| T-12-21 | Elevation of Privilege | Claude under-classified decision | mitigate | `test-risk-decisions.sh` includes a Claude-authored L2 decision containing `修改 JWT` and expects pending L3/L4 approval instead of Codex continuation. |
| T-12-22 | Repudiation | Approval replay | mitigate | `test-decision-replay.sh` asserts used, expired, project-mismatched, and task-mismatched approvals fail closed. |

</threat_model>
