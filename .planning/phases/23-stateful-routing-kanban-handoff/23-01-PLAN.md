```text
needs-user:
```

Required PM routing outcomes:

```text
question -> wait_for_user
needs_research -> create_research_task
requirement_ready -> create_tasks
feasibility_issue -> defer_to_human or block
```

```text
implementer.task_complete -> reviewer child task

reviewer.approved -> complete or QA child task
reviewer.findings -> implementer follow-up child task
```

```text
user-visible behavior change
cross-module or cross-boundary integration
high acceptance or regression risk
```

```text
new unverified stack/capability area
solution branching that changes downstream task graph
explicit research/comparison/proposal/feasibility request
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Role-engine response -> routing layer | External CLI outputs are interpreted into Kanban create/block/complete actions. |
| Routing helper -> Kanban metadata | Local orchestration writes machine-readable metadata that downstream tasks consume. |
| Human unblock -> resumed worker | User decisions can resume blocked work; routing must resume the right task or create the right child task. |
| Review/QA predicates -> task creation | Local predicates decide whether new cross-role tasks are inserted. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-23-01 | Tampering | Routing metadata schema | mitigate | Task 1 centralizes the four-field routing contract and tests for exact field presence. |
| T-23-02 | Spoofing | Block reason parsing | mitigate | Use fixed lightweight prefixes and shared parsing helpers instead of free-text keyword drift. |
| T-23-03 | Repudiation | Unblock/resume chain | mitigate | Same-role resumes and cross-role follow-ups are explicit and covered by smoke tests plus Kanban run history. |
| T-23-04 | Tampering | Comment-driven recovery | mitigate | Recovery logic reads normalized metadata and run summary/metadata, not scraped comments. |
| T-23-05 | Denial of Service | Parallel dispatcher entrypoints | mitigate | Preserve one public runtime entrypoint and migrate internals instead of running a second parallel dispatcher. |
| T-23-06 | Elevation of Privilege | Scope creep into risk hooks/timeout cleanup | mitigate | Plan explicitly excludes Phase 24/25 features from this phase's runtime changes. |
| T-23-07 | Information Disclosure | Cross-board leakage | mitigate | Preserve board-scoped execution and keep project isolation tests in the aggregate gate. |
</threat_model>
