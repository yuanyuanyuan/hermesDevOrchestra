## Threat Model

<threat_model>
Primary threats and required mitigations:

- **T10-01 User SOUL overwrite:** `setup.sh` overwrites upstream `~/.hermes/SOUL.md`. Mitigation: copy existing differing SOUL to `~/.hermes/SOUL.md.bak` before overwrite and do not churn backups on idempotent reruns.
- **T10-02 Skill path corruption:** Installing skills under the wrong nested path makes upstream Hermes unable to load them. Mitigation: copy each skill directly to `~/.hermes/skills/{skill-name}/` and verify each destination `SKILL.md`.
- **T10-03 PATH shadowing:** Creating a local `hermes` wrapper would invalidate the Phase 9 upstream boundary. Mitigation: create only `orch-*` helpers and explicitly avoid package bins or aliases named `hermes`.
- **T10-04 Privilege or installer escalation:** Running upstream installers, npm global installs, sudo, or transport-specific setup creates user-environment risk. Mitigation: preflight existing commands only; no `curl | bash`, no `npm install -g`, no sudo, no Telegram requirement.
- **T10-05 Event leakage/cross-project confusion:** Claude hooks writing only to a global event file loses project isolation. Mitigation: write every hook event to both `/tmp/hermes-orchestra/{project}/claude-events.jsonl` and `/tmp/hermes-orchestra/claude-events.jsonl`.
</threat_model>

Block on high-severity threats: if execution cannot preserve SOUL backup, direct skill layout, no-sudo behavior, or no local `hermes` wrapper, stop and record the blocker in `10-01-SUMMARY.md`.
