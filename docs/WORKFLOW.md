# Workflow

`WORKFLOW.md` is the workflow projection for the current root package.

Use these documents in order:

1. [GETTING-STARTED.md](GETTING-STARTED.md) for installation, initialization, and first-run setup.
2. [DEVELOPMENT.md](DEVELOPMENT.md) for day-to-day development and contribution flow.
3. [TDD-LONG-TASK-WORKFLOW.md](TDD-LONG-TASK-WORKFLOW.md) for test-first execution on larger implementation tasks.

Command entrypoints remain the local `orch-*` helpers under `scripts/bin/`.
