# Degradation Is Evidence State, Not Run Status

`degraded` is not a Six-Stage Run status. Run status remains `queued`, `running`, `blocked`, `failed`, `completed`, or `stopped`; degradation describes artifact, backend, projection, or evidence quality. The full system uses the state machine `normal -> degraded -> recovered` and `degraded -> blocked_due_to_degradation -> recovered`. Every degraded or recovered artifact must carry a `degradation_record` with class, cause, affected evidence refs, required decision, recovery options, acceptance ref, and completion-evidence policy. Degraded evidence does not satisfy required completion evidence by default; artifact-family exceptions must be declared in `config/degradation/policy.json` and accepted through the required Kimi or Human decision. Template debate fallback never counts as required debate coverage, and recovery writes replacement evidence instead of overwriting the degraded artifact.
